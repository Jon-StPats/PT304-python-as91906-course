# Checkpoints (Teacher + Student)

Use these as the "definition of done" for each folder.

## 01_basics
- [ ] Can run a Python file in VS Code terminal
- [ ] Uses variables (int/float/str/bool)
- [ ] Uses if/elif/else
- [ ] Uses for and while loops

## 02_functions_data
- [ ] Writes functions with parameters and return values
- [ ] Uses lists and dictionaries
- [ ] Iterates over collections

## 03_structured_programs
- [ ] Builds a loop-driven menu program
- [ ] Uses functions to keep code tidy
- [ ] Stores program data in a structured way

## 04_files
- [ ] Reads from a file
- [ ] Writes to a file
- [ ] Handles missing file (first run)
- [ ] Data persists between runs

## 05_classes
- [ ] Defines a class with attributes and methods
- [ ] Creates and uses objects
- [ ] Explains why classes improve structure

## 06_robustness_testing
- [ ] Validates user input
- [ ] Handles invalid input without crashing
- [ ] Completes a testing table with expected, boundary, invalid cases

## 07_project_prep
- [ ] Student can explain the two complex techniques they will use
- [ ] Student has a simple plan (features + data model)
- [ ] Student can identify at least 6 tests (2 expected, 2 boundary, 2 invalid)
